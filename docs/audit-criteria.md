# 审核标准

## 安全审核标准

### 🔴 Danger（不推荐发布）
- 安装命令指向非官方域名且无法验证来源
- 包名为随机字符串或与知名包名高度相似（疑似仿冒）
- 工具明确声明会将用户数据发送至第三方且无隐私政策
- 已知恶意软件分发渠道（参考：OpenClaw 2026-02 审计的 341 个恶意包）

### 🟡 Warn（可发布，需加说明）
- 非官方维护，维护者身份不明
- 涉及爬取行为（LinkedIn 抓取、竞品数据抓取等），存在 ToS 风险
- 需要 API Key 且 Key 权限范围较大
- 工具处于 alpha/beta 阶段，稳定性未知
- 涉及代理流量、中间人行为

### 🟢 Pass
- 官方维护（平台官方 / 知名公司）
- npm/pip 包名清晰，可在官方注册表验证
- 有明确的隐私政策或开源代码可审计
- stars 数较高（>500）且有持续维护记录

---

## 内容审核标准

### 🔴 必须修
- install_cmd 命令格式错误或与 README 不符
- 工具名称与实际功能严重不符
- 岗位分类明显错误（如把数据库工具分到内容创作）
- 关键字段为空（tagline / what / why）

### 🟡 建议修
- 夸大表述（「10倍效率」「秒变专家」「革命性」）且无数据支撑
- 同一岗位下多个工具的 tagline 风格差异过大
- example_biz 或 example_tech 过于抽象，缺乏具体场景
- tags 过多（>4个）或过少（0个）

### 🟢 通过
- 所有字段填写完整
- 描述准确、简洁、有场景感
- install_cmd 与官方文档一致

```markdown
### 🟡 建议修（N 项）

```
